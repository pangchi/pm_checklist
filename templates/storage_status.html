How Auto-Purge Works

After every photo upload and on server startup, disk usage on {{ partition }} is checked.

If usage is ≥ {{ threshold|int }}%, the oldest photos (by DB timestamp) are deleted from disk and their photo_path field is set to NULL in the database. Purging continues until disk usage falls below {{ (threshold - 5)|int }}% or no photos remain.

A Telegram alert is sent each time a purge occurs, listing the deleted files, bytes freed, and new disk usage.

Configure the threshold and monitored partition in config.ini under [storage].
